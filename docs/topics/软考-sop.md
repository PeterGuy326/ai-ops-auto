# 软考 运营 SOP

> 软考备考专题（系统架构师 / 高项 / 中项）。
> 本文件是 软考 专题的"业务事实底座"——给 LLM 当上下文，给 Topic Owner 当操作手册。
> 技术对抗（频率/指纹/养号）见 `docs/anti-risk.md`；平台改写风格见 `prompts/platform_style/*.md`。本文件不重复其内容。

## 1. 专题定位与边界

软考是面向"在职 IT 人员、技术管理者、央国企/事业单位在编人员"的资格考试备考专题，核心叙事是**"用工程师思维 + 真题驱动，把软考从'背 → 蒙 → 撞运气'变成可复现的通关流"**。范围严格限定在国家计算机软件资格考试（NCRE 软考），三个高优先级科目：

- **系统架构设计师**（高级，每年下半年 11 月一次）
- **信息系统项目管理师 / 高项**（高级，每年两次：5 月 + 11 月）
- **系统集成项目管理工程师 / 中项**（中级，每年两次，2024 年改革后并入"信息系统项目管理工程师"也归本专题）

判定 checklist（满足全部即"对题"）：

- [ ] 主语是软考三个目标科目之一（系架 / 高项 / 中项），不是泛"程序员考证"
- [ ] 必须有**真题引用 / 知识点出处**（年份 + 上午题号 / 下午题号 / 论文题号）
- [ ] 干货密度：≥ 60% 篇幅是可被截图保存的清单 / 表 / 思维导图，不是"我的备考心路历程"
- [ ] 不误导考点：所有论断必须能在教材或考试大纲（最新版）里找到出处；版本错误一律视为致命错误
- [ ] 不出现"包过 / 保过 / 押题命中率 99%"等违规宣传

跑题反例：
- ❌ "PMP 备考全攻略" —— 不是软考体系
- ❌ "考软考有什么用？" —— 选题层水帖，不在本专题做
- ❌ "我考了 10 个证" —— 偏离三科目聚焦

## 2. 受众画像

**主受众 A：在职 IT 工程师 / 技术骨干**

- 年龄 28-38，男性占 7:3，一二线 + 强二线占多数
- 职位：高级开发 / 架构师 / 技术经理；月薪 18-50k
- 痛点：① 公司晋升 / 评职称 / 入户加分 / 央国企招聘前置门槛；② 一边上班一边备考时间紧；③ 教材厚（系架 1500+ 页）看不下去；④ 论文写不出来卡死
- 高频搜索词："系统架构师 通过率"、"系架 论文 写法"、"高项 真题 解析"、"软考 报名 时间"、"系统架构师 知识点 整理"、"软考 答题 技巧"
- 活跃时段：工作日 12:30-13:30 / 21:00-24:00；周末 9:00-12:00 / 14:00-17:00
- 平台偏好：知乎（深度备考方法论）= 公众号（订阅备考号）> B 站（看视频精讲）> 小红书（图文笔记）> 头条（少）

**主受众 B：央国企 / 事业单位 / 体制内技术岗**

- 年龄 30-45，男女比 5:5，二三线城市为主
- 痛点：① 单位强制要求；② 评聘高级职称必备；③ 不熟悉互联网学习渠道；④ 习惯纸质资料、视频精讲
- 高频搜索词："软考 评职称"、"高项 论文模板"、"信息系统项目管理师"、"软考 PDF 资料"、"软考 答题卡 填涂"
- 活跃时段：工作日 8:30-9:30 / 17:30-18:30；午休 12:00-13:30；周末白天
- 平台偏好：公众号 > 头条 > 知乎 > B 站 > 小红书

**主受众 C：应届 / 准备考研失败转就业 / 想加户口加分**

- 年龄 22-26，地域分布广
- 痛点：① 一线城市落户加分（上海、广州等地高项可加分）；② 简历背书；③ 经济敏感（不愿意报上万的培训班）
- 高频搜索词："软考 含金量"、"软考 入户"、"系架 vs 高项"、"软考 自学"、"软考 免费资料"
- 平台偏好：小红书（图文笔记 + 9 图卡片）> B 站 > 知乎 > 头条 > 公众号

## 7. 发布节奏 SOP

**频率上限**：严格遵循 `docs/anti-risk.md` 全局规则——单账号每日 ≤ 2 条、最小间隔 4 小时、养号期 7-14 天内不发布、凌晨 0-6 点禁发。本节只写软考专题的差异化节奏。

**软考专题最佳时段**（备考人群作息）：

| 平台 | 工作日窗口 | 周末窗口 | 备注 |
|---|---|---|---|
| 公众号 | 7:30-8:30 / 12:30-13:30 / 21:00-23:00 | 8:30-11:30 / 21:00-23:00 | 备考人群在通勤 / 午休 / 睡前学习时段集中 |
| 知乎 | 12:30-13:30 / 21:00-24:00 | 9:00-12:00 / 21:00-24:00 | 周末晚是知乎备考问答高峰 |
| B 站 | 19:00-23:00 | 9:00-12:00 / 14:00-17:00 / 19:00-23:00 | 视频投递周二/周四晚 21:00 最优 |
| 小红书 | 12:30-13:30 / 19:00-22:30 | 同左 | 学习类内容在周末上午也有强转化 |
| 头条 | 7:00-9:00 / 17:30-19:30 | 9:00-12:00 / 17:00-20:00 | 体制内中年用户的早高峰是关键 |

**周期性节奏**（这是软考与 dws / 篮球的最大差异）：

软考有明确的"考试周期"，发布节奏必须跟考试日历对齐：

| 时段 | 距离考试 | 内容重心 | 发布密度 |
|---|---|---|---|
| 报名期（考前 3-4 个月） | -120 天 | 报名指南 / 科目选择 / 备考规划 | 每周 +30% |
| 备考前期（-120 ~ -60） | 系统性长文 / 教材精读 | 每周常规节奏 |
| 备考中期（-60 ~ -30） | 真题精讲 / 错题复盘 / 论文模板 | 每周 +20% |
| 冲刺期（-30 ~ -7） | 考点速通 / 押题（合规版）/ 论文模板 | 每周 +50%，B 站短视频日更 |
| 考前 7 天 | 考场流程 / 答题技巧 / 心态调整 | 每日 1 条轻内容 |
| 考试当日 - 当周 | 真题回忆 / 答案对照 / 估分指南 | 当日 / 当周冲一波，是年度流量峰值 |
| 出分后 | 通过率分析 / 下一轮规划 | 每周 1 条 |
| 淡季 | 知识点系列 / 长尾 SEO | 每周常规节奏 |

**每周条数建议**：

- 公众号：每周 2-3 条（冲刺期可冲到 4-5 条）
- 知乎：每周 1-2 条长文 + 5-10 条问答
- B 站：每周 1-2 条长视频 + 3-5 条短视频
- 小红书：每周 3-5 张图文卡片（备考类高契合，可以稍密）
- 头条：每周 3-5 条

**节假日**：春节、五一、国庆是备考黄金期，反而要加密发布（与 dws 相反）；考试结束当周是流量高峰，必须排好年度日历提前布稿。

**时间打散**：必须 `+ random(0, 600)s`，禁止整点发布——见 `docs/anti-risk.md` 第四层"行为层"。

## 8. 合规与红线

**通用红线**：
- ❌ **不能写"包过 / 保过 / 押题命中率 90%"等违规宣传**——属于教育广告法红线，账号会被一键封停
- ❌ 不出售 / 转发盗版教材 PDF（"备考资料包"必须是自整理的清单 / 思维导图，不是教材原文扫描）
- ❌ 不假冒"软考官方 / 工信部 / 中国计算机技术职业资格网"任何官方名义
- ❌ 不出售真题（真题在考完后可公开讨论，但考前以"押题"名义售卖违规）
- ❌ 不发布"代考 / 替考 / 找关系" 等内容（刑事红线）

**考点引用规范**：
- 教材版本：必须以最新版清华大学出版社官方教材为准，且在文章里标注"本文基于 XXXX 版教材"
- 大纲版本：必须以工信部 / 软考办最新发布版本为准，引用大纲条目时带版本号
- 真题来源：标注年份 + 半年（上 / 下）+ 题型（上午 / 下午 / 论文）+ 题号
- 错题修正：发现自己写错或教材有勘误，必须在置顶评论 / 文末标注，不能默默删稿

**禁用词**：
- "包过" / "保过" / "稳过" / "0 基础轻松过" / "捷径" / "速成"（违规宣传）
- "工信部内部" / "考试院流出"（涉嫌假冒官方）
- "学不会退钱"（商业承诺需走合同，不在内容里随口许诺）

**软考专属红线**：
- 不晒"考前一天找关系拿到的题"（即使是段子，也会引发举报）
- 不发布"用某某软件作弊"内容
- 论文模板必须明确标注"仅供参考、考试需结合个人项目"，不能让读者照抄

## 9. KPI 与异常处置

**健康基线**（软考专题，受众更窄但更精准，互动率应高于泛账号）：

| 平台 | 单篇 1h 曝光中位数 | 24h 数据 | 互动指标 |
|---|---|---|---|
| 公众号 | ≥ 800 | 阅读完成率 ≥ 30% | 收藏 / 阅读 ≥ 5%（备考内容收藏高） |
| 知乎 | ≥ 1500 长文 / 3000 问答 | 收藏率 ≥ 6%（关键指标） | 赞同 ≥ 50 |
| B 站 | ≥ 3000 播放 | 完播率 ≥ 40%（备考视频高完播） | 三连率 ≥ 5% |
| 小红书 | ≥ 1000 | 收藏率 ≥ 8%（考点卡片收藏极高） | 互动 ≥ 5% |

| 信号 | 阈值 | 处置 |
|---|---|---|
| 24h 曝光 < 中位数 20% | 单条 | 标记 DEGRADED；同账号暂停 48h；下一条改"政策解读"类安全选题 |
| 公众号"广告违规" / "教育广告"提示 | 立即 | 下架 + 改写删去违规词；24h 不再发；走申诉 |
| 知乎"内容审核中" > 12h | 单条 | 检查是否有"包过 / 押题"等敏感词；轻量改写后投另一账号 |
| B 站视频"创作中心提示考虑修改" | 单条 | 检查是否标题党 / 真题片段是否涉版权 |
| 小红书笔记审核未通过 | 单条 | 检查"考前 X 天"等时效词是否触发"不实信息"判定 |
| 评论区出现 ≥ 3 条"举报代考"投诉 | 单条 | 立即排查内容；如果是误伤要置顶澄清；如果命中红线立即删除 |

**异常处置 SOP**：
1. 触发后立刻 `update_health(DEGRADED)`（参考 `docs/anti-risk.md` 链路），暂停 48h
2. 复盘三件事：违规词是否触线、教材版本引用是否过期、真题年份是否标错
3. 复盘记录写到软考知识库 `events/yyyy-mm-dd-account-degraded.md`
4. 48h 后恢复，第一条必须是"学习方法 / 心态调整"等纯安全选题，不要立刻碰"押题 / 真题"
5. 出分季节（每年 12 月 + 6 月）流量峰值期前 2 周必须排查所有历史内容里是否有"包过"残留措辞

## 10. 好坏样本对比

### 好样本 A（知乎长文）

> **标题**：系统架构师 2024 下半年真题（下午案例 5 道）全解 + 答题模板
>
> **开头**：本文基于 2024 下半年系架下午案例 5 道真题（题目编号 1-5，原题文末附完整 PDF），逐题给出考点出处、答题模板、扣分陷阱。所有解析基于清华出版社 2023 版《系统架构设计师教程》第 4 版，2024 版补充章节已对照标注。这套答题模板我用了 30 天，第二次冲刺通过。

**为什么好**：① 主语精准（系架真题）；② 教材版本明示（4 版 + 2024 补充）；③ 题目年份标到半年级（2024 下半年）；④ 给"答题模板 + 真题 PDF"的明确收益预期；⑤ 自带身份（"通过的人"，建立可信度）。

### 好样本 B（小红书 9 图卡片）

> **标题**：💡 系架软件架构风格 9 图整理｜每年必考
>
> **第 1 图**：背景 + 每年必考的真实统计：2018-2024 共 7 年，"软件架构风格"在系架上午题出现 12 次（截图标注题号）。
>
> **后 8 图**：每图一种架构风格（管道-过滤器 / 数据抽象 / 事件驱动 / 仓库结构 / 解释器 / 客户/服务器 / 分层 / 黑板），每图含定义 + 优缺点 + 一道真题示例。

**为什么好**：① 选题颗粒度刚好（一个章节深挖）；② 数据驱动（"7 年 12 次"）；③ 9 图卡片是小红书的备考爆款结构；④ 标题里"必考"是合规版的关键词（不是"包过"）；⑤ 收藏率会拉满，给账号灌权重。

### 坏样本 A（违规红线）

> **标题**：🔥 软考高项内部押题命中率 95%，速领！
>
> **问题**：① "押题命中率 95%" 触发广告法（虚假宣传）；② "速领" + "内部"是公众号 / 知乎双高敏关键词；③ 一条内容触发限流是次要的，主要是账号会进入"教育广告违规"观察期，连带影响整月流量；④ 对真备考用户也是反作用——真备考人识破后会取关。

### 坏样本 B（教材版本翻车）

> **标题**：系架"软件架构风格"知识点整理
>
> **问题（隐性）**：作者引用了 2018 年的旧版教材，但 2023 年第 4 版教材已经修订了部分定义（例如把"客户/服务器"明确独立为一条风格）。考生按这篇复习，考场上踩坑会归罪到博主头上——评论区被举报"误导考点"，账号"专业度"权重立刻塌方。**软考专题的所有内容都必须标注教材版本**，这是和 dws / 篮球最大的合规差异。

### 坏样本 C（跑题水帖）

> **标题**：考完软考的一些感受
>
> **问题**：① 没有考点、没有真题、没有方法论；② 受众读完拿不走任何动作；③ 算法侧会判为"水内容"，权重稀释；④ LLM 喂这种内容生成下一篇会继续"心路历程化"，损害整个专题的事实底座质量。

> 本 SOP 是软考专题 LLM 生成 prompt 的事实底座。每年考试改革（大纲版本 / 教材版本 / 报名规则）后，必须更新第 1 / 第 8 章；考点解析类内容必须以更新后的版本为准。

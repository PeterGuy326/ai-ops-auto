# dws 运营 SOP

> 钉钉企业能力（AI 表格 / 审批 / 听记 / IM 机器人 / 日历 / 待办 / 钉钉文档）专题。
> 本文件是 dws 专题的"业务事实底座"——给 LLM 当上下文，给 Topic Owner 当操作手册。
> 技术对抗（频率/指纹/养号）见 `docs/anti-risk.md`；平台改写风格见 `prompts/platform_style/*.md`。本文件不重复其内容。

## 1. 专题定位与边界

dws 是面向"使用钉钉作为日常协作底座的企业用户与开发者"的能力专题，核心叙事是**"把钉钉做成可被 LLM/Agent 直接驱动的协作中枢"**，覆盖 AI 表格、审批、听记纪要、IM 机器人、日历、待办、钉钉文档、知识库、钉钉云盘等具体能力。受众读完一篇内容，应当能**立刻在自己钉钉里复刻一个动作**——而不是停留在"这玩意儿听上去很酷"。

判定 checklist（满足全部即"对题"）：

- [ ] 主语是"钉钉的某个具体能力"（AI 表格 / 审批 / 听记 / 机器人 / 日历 / 待办 / 文档 / 云盘 / 知识库 / 在线电子表格），不是泛泛的"协作工具"
- [ ] 落到一个具体的协作场景（OKR 同步 / 周报汇总 / 会议纪要分发 / 审批流转 / 客户跟进），不是空谈"提效"
- [ ] 有可被复现的"5 分钟动手做"或"一句命令搞定"片段，最少给一个截图位或步骤清单
- [ ] toB 调性：第三人称或"我们团队"为主，不出现"姐妹"/"宝子"/"集美"这类小红书 C 端口语
- [ ] 关键能力名以钉钉官方命名为准（"AI 表格"不写成"智能表格"，"听记"不写成"会议录音"），与 dws skill 描述一致

跑题反例：
- ❌ "AI 时代的协作软件趋势" —— 没有任何具体钉钉能力
- ❌ "飞书/企微/钉钉哪家强" —— 横向对比对 dws 是稀释而非聚焦
- ❌ "我用钉钉摸鱼的 10 个技巧" —— 调性与企业决策者冲突

## 2. 受众画像

**主受众 A：企业 IT / 信息化负责人 / 协同平台运营**

- 年龄 30-45，男女比 6:4，集中在制造、零售、教育、医疗、政府事业单位
- 城市分布：长三角 / 珠三角 / 川渝 / 京津冀，二线城市占比远高于一线（钉钉用户结构特征）
- 收入：年薪 25-60 万，决策权限 5-50 万级 SaaS 采购
- 痛点：领导要 AI 化但预算紧、跨部门数据壁垒、审批流程长、纪要靠人记不全、机器人想做但 IT 资源不够
- 高频搜索词："钉钉 AI 表格 怎么用"、"钉钉机器人 webhook"、"钉钉审批 自定义"、"钉钉听记 导出"、"钉钉 集成 ChatGPT"
- 活跃时段：工作日 9:00-11:00 / 14:00-17:30；周末几乎不刷
- 平台偏好：公众号（订阅 IT/数字化转型号）> 知乎（搜索决策依据）> 头条（碎片浏览）> B 站（看演示视频）> 小红书（极少）

**主受众 B：钉钉生态开发者 / ISV / 个人插件作者**

- 年龄 24-35，男性占主导（≥ 8:2），一线 + 新一线为主
- 痛点：开放平台文档碎片、SDK 版本迭代快、AI 表格 API 限频踩坑、企业内部审计要求严
- 高频搜索词："钉钉开放平台 API"、"AI 表格 字段类型"、"钉钉机器人 webhook 签名"、"stream 模式"、"dws"
- 活跃时段：工作日 10:00-12:00 / 21:00-23:30；周末晚上活跃
- 平台偏好：知乎（深度技术文）> 公众号（订阅技术号）> B 站（看实操视频）> 头条（少）> 小红书（少）

**次要受众 C：中层管理者 / 业务一线**

- 不写代码、但管表格管审批，是 AI 表格和审批自动化的最终用户代表
- 容易被"一键生成周报"、"3 分钟搭一个审批"这类成果型内容吸引
- 平台偏好：小红书（图文卡片）+ 公众号

| 平台 | 优先级 | 契合度（1-5） | 该专题策略 |
|---|---|---|---|
| 公众号 | 主攻 | 5 | toB 决策链阅读习惯首选；放标杆案例长文、能力解读、版本动态 |

## 7. 发布节奏 SOP

**频率上限**：严格遵循 `docs/anti-risk.md` 全局规则——单账号每日 ≤ 2 条、最小间隔 4 小时、养号期 7-14 天内不发布、凌晨 0-6 点禁发。本节只写 dws 专题的差异化节奏。

**dws 专题最佳时段**（toB 调性专属）：

| 平台 | 工作日窗口 | 周末窗口 | 备注 |
|---|---|---|---|
| 公众号 | 7:30-8:30（通勤）/ 12:30-13:30（午休）/ 21:00-22:00（睡前充电） | 9:00-11:00 | 周一最佳，周五次之，周末转化弱 |
| 知乎 | 12:00-13:30 / 21:00-23:00 | 10:00-12:00 / 21:00-23:00 | 周末晚上是阅读高峰 |
| 头条 | 7:00-9:00 / 18:00-20:00 | 9:00-11:00 / 19:00-21:00 | 中老年用户多，早高峰是关键 |
| 小红书 | 12:30-14:00 / 19:00-22:00 | 同左 | toB 内容慎选窗口，避开 22:00 后女性活跃高峰 |
| B 站 | 19:00-22:00 | 14:00-17:00 / 19:00-22:00 | 视频投递最好周二 / 周四晚 |

**每周条数建议**：

- 公众号：每周 2-3 条，深度长文为主，节奏不能比月更慢
- 知乎：每周 1-2 条长文 + 3-5 条问答，问答可以加速账号活跃度
- 头条：每周 3-5 条，可与公众号同源改写
- 小红书：每周 ≤ 2 条，避免频繁低契合内容拉低号权重
- B 站：每月 2-4 条视频，质量优先

**节假日策略**：

- 春节 / 国庆长假：暂停 toB 内容（决策者不在线），改投"年度盘点 / 复盘"类长文
- 国庆 / 五一前 1 周：是"年度规划 / 数字化预算"内容的黄金窗口
- 周五下午 17:00 后到周日：发布转化全平台明显下降，公众号尤甚

**时间打散**：必须 `+ random(0, 600)s`，禁止整点发布——见 `docs/anti-risk.md` 第四层"行为层"。

## 8. 合规与红线

**通用红线**：
- 不写"钉钉 vs 飞书 / 企微"的拉踩对比——可以做能力对比，不带价值判断（"钉钉 X 不如飞书 Y"这种句式禁用）
- 不暴露任何客户企业名称（除非客户书面授权放案例）
- 不放真实员工头像 / 真实群聊截图（必须打码）
- 商业合作需在文首明示"# 推广 / # 商业合作"，公众号需挂"广告"标识

**专业领域引用规范**：
- 引用钉钉官方能力名以最新开放平台文档为准（`https://open.dingtalk.com/`），术语不自创
- 引用客户数据需注明区间和口径："某零售企业 30 天数据"而非"某客户"
- 引用第三方研究需带来源（艾瑞 / IDC / 信通院 等），且发布不超过 18 个月

**禁用词**：
- "破解" / "绕过钉钉风控" / "免审核" / "白嫖" —— 与企业级合规调性冲突
- "你必须" / "你应该" / "再不学就晚了" —— 说教语气
- 涉及竞品的负面修饰词（"垃圾" / "拉胯" / "智商税"）

**dws 专属红线**：
- 不晒"如何用钉钉机器人监控员工"类内容（隐私合规高敏）
- 不展示企业组织架构图全貌（防止社工攻击切入点）
- 不分享会涉及鉴权 secret / corp_id 的截图

| 信号 | 阈值 | 处置 |
|---|---|---|
| 24h 曝光 < 中位数 20% | 单条 | 标记 DEGRADED；同账号暂停 48h；下一条改用"案例长文"安全选题 |

**异常处置 SOP**：
1. 触发后立刻 `update_health(DEGRADED)`（参考 `docs/anti-risk.md` 的 health 链路），暂停 48h
2. 复盘三件事：选题是否跑题、标题是否标题党、节奏是否撞红线
3. 复盘记录写到 dws 知识库 `events/yyyy-mm-dd-account-degraded.md`
4. 48h 后恢复发布，第一条必须是"低风险类型"（案例复盘 / 数据盘点），不要再上"机器人 / 接口"类高敏内容

## 10. 好坏样本对比

### 好样本 A（公众号长文，预估打开率 ≥ 6%）

> **标题**：把例会纪要交给钉钉听记｜30 人项目周会的 30 天实录
>
> **开头**：上周一例会跑完，我们没安排实习生记纪要。30 分钟会议结束 3 分钟后，AI 听记把要点甩进了项目群，自动生成 12 条待办分派给具体责任人，跨部门同事在群里发"+1"确认接收。这套流，我们试了 30 天，今天把链路全公开。
